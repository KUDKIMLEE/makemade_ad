# makemade 검색광고 순위 추적기

네이버 검색광고(파워링크) 키워드별 **평균노출순위(avgRnk)** 를 조회하는 Vercel용 대시보드입니다.
비밀키는 서버리스 함수 안에서만 사용되고 브라우저로 노출되지 않습니다.

## 구조
```
naver-rank-tracker/
├── api/rank.js     # 네이버 API 호출 (서버리스 함수)
├── index.html      # 대시보드 화면
├── .env.example    # 환경변수 템플릿
└── README.md
```

## 1. API 키 발급
1. 네이버 검색광고(searchad.naver.com) 로그인
2. **도구 → API 사용 관리**에서 액세스라이선스 · 비밀키 발급
3. 우측 상단 계정 정보에서 **고객 ID(CUSTOMER_ID)** 확인 (숫자)

## 2. 배포 (Vercel)
1. 이 폴더를 GitHub 저장소로 올리거나, `vercel` CLI로 배포
2. Vercel 프로젝트 → **Settings → Environment Variables** 에 아래 3개 등록
   - `NAVER_API_KEY` (액세스라이선스)
   - `NAVER_SECRET_KEY` (비밀키)
   - `NAVER_CUSTOMER_ID` (고객 ID)
3. 다시 배포하면 `/api/rank` 가 활성화됩니다.

> Vercel은 루트의 `index.html`을 정적 페이지로, `api/` 폴더를 서버리스 함수로 자동 인식합니다. 별도 빌드 설정이 필요 없습니다.

## 3. 사용
- 대시보드에는 네이버 내보내기 기준 **84개 키워드가 이미 등록**되어 있습니다 (`keywords.json`, `index.html`에 내장).
- 검색창·빠른 필터(티셔츠/공장/다이마루 등)·정렬(순위·노출·가나다)로 관리합니다.
- 날짜 선택 후 **순위 조회**를 누르면, 프론트가 등록된 키워드 ID를 백엔드로 보내고 백엔드는 `/stats`만 호출해 평균순위를 채웁니다. (캠페인 순회가 없어 빠릅니다.)

배포 전 미리보기에서는 백엔드가 없으므로 **데모 데이터**가 표시됩니다.

### 순위 추이 (그래프)
- 기간(7·14·30일)을 고르고 **추이 조회**를 누르면, 백엔드가 날짜별로 `/stats`를 호출해 각 키워드의 일별 평균순위·노출수를 가져옵니다. 저장 없이 네이버 일별 데이터를 그대로 그리므로 최대 92일까지 가능합니다.
- 목록 각 행에 순위 **스파크라인**과 기간 시작 대비 변화(초록 세모=상승)가 표시되고, 행을 누르면 **순위 추이 라인 차트 + 노출수 막대 차트**가 열립니다. 순위축은 위가 상위(1위)입니다.
- 30일처럼 긴 기간은 호출이 많아 몇 초 걸릴 수 있습니다. Vercel 무료 플랜의 함수 타임아웃이 짧으면 `vercel.json`으로 `maxDuration`을 늘리거나 기간을 14일 이하로 쓰세요.

### 키워드 목록 갱신
네이버에서 키워드를 추가/삭제하면 검색광고 → 키워드 내보내기(xlsx)를 다시 받아 `keywords.json`을 갱신하면 됩니다. `index.html` 안의 `const KEYWORDS = [...]` 배열도 같은 내용으로 바꿔주세요(미리보기용 내장 데이터).

## 알아둘 점
- **전일 기준 통계**입니다. 당일 실시간 순위는 공식 API로 제공되지 않습니다.
- **내 광고 계정의 키워드만** 조회됩니다. 경쟁사 순위는 볼 수 없습니다.
- 실제 노출 순위는 입찰가 × 품질지수, 그리고 검색하는 사람의 지역·기기·시간대에 따라 실시간으로 달라집니다. 평균노출순위는 추세 파악용 지표로 보는 것이 정확합니다.
- 키워드가 많으면 캠페인→광고그룹→키워드를 순회하느라 응답이 느려질 수 있습니다. 필요하면 결과를 캐싱하거나 키워드 ID를 미리 저장해 두는 방식으로 최적화할 수 있습니다.

## 자동화(선택)
매일 아침 자동 기록을 원하면 Vercel Cron으로 `/api/rank`를 호출해 결과를 DB(예: Vercel KV, Supabase)에 저장하면 날짜별 순위 추이 그래프까지 만들 수 있습니다.
